jddesvr - Java интерфейс к DDE-серверу обработки XLTable+POKE запросов.

Сначала необходимо выполнить генерацию java классов и кода jni. Для
этого нужно в каталоге проекта выполнить команду ant, в результате
чего в ../build появятся файлы jddesvr.jar и jddesvr-javadoc.jar,
а в каталоге проекта появится (или обновится) файл jddesvr_wrap.cxx,
содержащий код для генерации jni библиотеки.

Сборка dll-библиотеки должна выполняться только при наличии файла
jddesvr_wrap.cxx. После сборки в ../build появится файл jddesvr.dll,
который используется совместно с jddesvr.jar.

Важно:
1. При наследовании ServiceHandler в java не забыть конструктор
наследника с вызовом конструктора базового класса. Например

class MyCallback extends ServiceHandler {

	public MyCallback(String serviceName) {
		super(serviceName);
	}
    ...

2. На стороне Java важно сохранять ссылку на экземпляр ServiceHandler
до тех пор, пока ожидается обработка событий. Когда обработка событий
больше не требуется обязательно удалять сервис из сервера. Для более
безопасного использования данного адаптера лучше всего сделать еще один
враппер уже на jddesvr::Server с сохранением ссылок на все зарегистрированные
обработчики сервисов.

3. Нельзя вызывать методы сервера из обработчиков.

4. В обработчик событий ServiceHandler добавлен метод

bool onRawData(string topic, string item, char* data, int size)

Этот метод вызывается в момент получения транзакции POKE перед
формированием ХlTable. Данный метод предназначен для использования в
случаях, когда необходимо работать с "сырыми" Xlt-данными. Обработчик
onRawData должен возвращать true, если обработка была выполнена и
вызывать метод onData не требуется. По умолчанию этот метод возвращает
false.
